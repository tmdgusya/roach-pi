Engineering Discipline Extension 엔지니어링 디사이플린 익스텐션

Strict engineering discipline and agentic orchestration for the pi coding agent. Multi-agent workflows, executable plans, and milestone-level decomposition. pi 코딩 에이전트를 위한 엄격한 엔지니어링 디사이플린과 에이전트 오케스트레이션. 다중 에이전트 워크플로, 실행 가능한 계획, 마일스톤 분해.

● Why ROACH PI? 왜 ROACH PI인가요?

●
Fully Open Source 완전한 오픈소스

Every line is on GitHub. No hidden prompts, no secret system instructions, no obfuscated behavior. Tools, event hooks, skill injections, and agent prompts are all plain TypeScript and Markdown.

모든 코드가 GitHub에 공개되어 있습니다. 숨겨진 프롬프트, 비밀 시스템 지시어, 난독화된 동작 없음. 도구, 이벤트 훅, 스킬 주입, 에이전트 프롬프트가 모두 일반 TypeScript와 Markdown으로 작성되어 있습니다.

★
Observable 옵저버빌리티

The footer displays real-time metrics every session: prompt cache hit rate, context usage bar, active tools, branch, and model. See exactly how your context is being utilized.

푸터에 매 세션 실시간 메트릭이 표시됩니다: 프롬프트 캐시 적중률, 컨텍스트 사용량 바, 활성 도구, 브랜치, 모델. 컨텍스트가 어떻게 사용되는지 정확히 확인하세요.

■
Transparent by Design 디자인부터 투명하게

No magic. Read the source and see exactly what the agent does. Skill rules are plain Markdown. Agent prompts are plain text. Every event hook is auditable.

마법 없음. 소스를 읽고 에이전트가 정확히 무엇을 하는지 확인하세요. 스킬 규칙은 일반 Markdown, 에이전트 프롬프트는 일반 텍스트, 모든 이벤트 훅이 감사 가능합니다.

▶
Simple Over Clever 화려함보다 단순함

We prefer straightforward solutions over elaborate orchestration. Fewer moving parts, fewer failure modes. See our contributing guidelines.

복잡한 오케스트레이션보다 단순한 해결책을 선호합니다. 움직이는 부분이 적을수록 실패도 적습니다. 기여 가이드라인을 참조하세요.

▶ Quick Start 빠른 시작

The extension provides a three-phase workflow: clarify → plan → execute. Each phase is driven by a slash command.

익스텐션은 3단계 워크플로를 제공합니다: 명확화 → 계획 → 실행. 각 단계는 슬래시 명령으로 구동됩니다.

  1. /setup — Configure Settings — 설정 구성

    Run once after installation to apply recommended settings. Sets quietStartup: true so the extension's custom ROACH PI banner replaces the default startup listing.

    설치 후 한 번 실행하여 권장 설정을 적용합니다. 익스텐션의 커스텀 ROACH PI 배너가 기본 시작 목록을 대체하도록 quietStartup: true를 설정합니다.

  2. /clarify — Resolve Ambiguity — 모호성 해결

    The agent asks dynamic, context-aware questions one at a time while exploring the codebase in parallel. Ends with a structured Context Brief that defines scope, constraints, and success criteria.

    에이전트가 코드베이스를 병렬로 탐색하면서 동적이고 문맥에 맞는 질문을 하나씩 묻습니다. 범위, 제약, 성공 기준을 정의하는 구조화된 컨텍스트 브리프로 끝납니다.

  3. /plan — Create an Executable Plan — 실행 가능한 계획 생성

    Transforms the Context Brief into an implementation plan with exact file paths, code blocks, commands, and expected outputs. No placeholders — every step is immediately executable by a worker agent.

    컨텍스트 브리프를 정확한 파일 경로, 코드 블록, 명령어, 예상 출력이 포함된 구현 계획으로 변환합니다. 플레이스홀더 없음 — 모든 단계가 워커 에이전트가 즉시 실행할 수 있습니다.

  4. /ultraplan — Milestone Decomposition — 마일스톤 분해

    For complex multi-day tasks. Dispatches 5 independent reviewer agents in parallel (Feasibility, Architecture, Risk, Dependency, User Value), then synthesizes findings into an optimized milestone DAG.

    복잡한 며칠짜리 작업을 위해. 5개의 독립적인 리뷰어 에이전트를 병렬로 디스패치(타당성, 아키텍처, 리스크, 의존성, 사용자 가치)한 후, 최적화된 마일스톤 DAG로 종합합니다.

  • /loop — Session Loop Scheduler — 세션 루프 스케줄러

    Schedule recurring prompts at fixed intervals. Cron-style timing — fires on schedule regardless of execution state. Supports 5s, 10m, 2h, 1d formats. Up to 100 concurrent jobs with per-job error isolation and automatic cleanup on session shutdown.

    고정 간격으로 반복 프롬프트를 스케줄링합니다. Cron 스타일 타이밍 — 실행 상태와 관계없이 스케줄대로 실행됩니다. 5s, 10m, 2h, 1d 형식을 지원합니다. 최대 100개 동시 작업, 작업별 에러 격리, 세션 종료 시 자동 정리.

    • 💡 Tip: The ask_user_question tool is always available — the agent will ask clarifying questions autonomously whenever it detects ambiguity, even outside /clarify mode.

    💡 팁: ask_user_question 도구는 항상 사용 가능합니다 — /clarify 모드 외에서도 모호성을 감지하면 자동으로 질문합니다.

    ■ Key Features 주요 기능

    ★
    Multi-Agent Orchestration 다중 에이전트 오케스트레이션

    12 bundled agents for exploration, execution, planning, and review. Delegate tasks via single, parallel (max 8 tasks, 4 concurrent), or chain modes with cycle detection and depth guards.

    탐색, 실행, 계획, 리뷰를 위한 12개 번들 에이전트. 사이클 감지 및 깊이 가드와 함께 단일, 병렬(최대 8개 작업, 4개 동시), 체인 모드로 작업을 위임하세요.

    ●
    Engineering Discipline 엔지니어링 디사이플린

    Karpathy rules auto-injected into code-writing agents. Automatic slop-cleaner spawns after successful runs to remove LLM-specific patterns while preserving behavior.

    Karpathy 규칙이 코드 작성 에이전트에 자동 주입됩니다. 성공적인 실행 후 동작을 유지하면서 LLM 특정 패턴을 제거하는 자동 슬롭 클리너가 실행됩니다.

    ■
    Plan Execution Pipeline 계획 실행 파이프라인

    plan-compliance → plan-worker → plan-validator pipeline with information barriers. Validators never see execution context — they judge solely from the plan document and codebase.

    정보 장벽이 있는 plan-compliance → plan-worker → plan-validator 파이프라인. 검증자는 실행 컨텍스트를 보지 않습니다 — 계획 문서와 코드베이스만으로 판단합니다.

    ▶
    TUI Customization TUI 커스터마이제이션

    Custom ROACH PI ASCII banner, branded footer with directory, branch, model, context bar, cache hit rate, and active tools display.

    디렉토리, 브랜치, 모델, 컨텍스트 바, 캐시 적중률, 활성 도구 표시가 있는 커스텀 ROACH PI ASCII 배너 및 브랜디드 푸터.

    ●
    Context Management 컨텍스트 관리

    Microcompaction truncates old tool results. Phase-aware summarization preserves workflow state across compaction. Phase and goal document survive context limits.

    마이크로컴팩션이 오래된 도구 결과를 자릅니다. 단계 인식 요약이 컴팩션 전반에 걸쳐 워크플로 상태를 보존합니다. 단계 및 목표 문서가 컨텍스트 한계를 초과합니다.

    ▶
    Session Loop Scheduler 세션 루프 스케줄러

    Schedule recurring prompts with /loop 5m check status. Cron-style fixed intervals, per-job error isolation, AbortController cancellation, timeout protection, and automatic cleanup on session shutdown. Up to 100 concurrent jobs.

    /loop 5m check status로 반복 프롬프트를 스케줄링합니다. Cron 스타일 고정 간격, 작업별 에러 격리, AbortController 취소, 타임아웃 보호, 세션 종료 시 자동 정리. 최대 100개 동시 작업.

    ★
    11 Behavioral Skills 11개 행동 스킬

    Bundled LLM skill rulesets: clarification, plan crafting, run plan, review work, simplify, systematic debugging, Karpathy discipline, Rob Pike optimization, and more.

    번들 LLM 스킬 룰셋: 명확화, 계획 작성, 계획 실행, 작업 리뷰, 단순화, 체계적 디버깅, Karpathy 디사이플린, Rob Pike 최적화 등.

    ◆
    FFF Search Engine FFF 검색 엔진

    Built-in search acceleration for find and grep through a native FFF engine. Adds multi_grep, frecency-aware ranking, git-aware indexing, and optional replacement of @ file autocomplete.

    네이티브 FFF 엔진을 통한 find / grep 내장 검색 가속. multi_grep, frecency 기반 랭킹, git 인식 인덱싱, 선택적 @ 파일 자동완성을 추가합니다.

    ● FAQ

    What is the pi coding agent? pi 코딩 에이전트란 무엇인가요?

    pi is a terminal-based AI coding agent that helps you read, write, and debug code directly from your terminal. It supports extensions, custom tools, themes, and multiple AI providers. GitHub →

    pi는 터미널에서 직접 코드를 읽고, 쓰고, 디버깅하는 터미널 기반 AI 코딩 에이전트입니다. 익스텐션, 커스텀 도구, 테마 및 여러 AI 프로바이더를 지원합니다. GitHub →

    Do I need to install the engineering-discipline skills separately? 엔지니어링 디사이플린 스킬을 별도로 설치해야 하나요?

    No. All engineering discipline skills are bundled with this extension. Just install the extension and run /setup — everything is ready out of the box.

    아니요. 모든 엔지니어링 디사이플린 스킬이 이 익스텐션에 번들로 포함되어 있습니다. 익스텐션을 설치하고 /setup을 실행하세요 — 모든 것이 즉시 사용 가능합니다.

    How do the subagent modes work? 서브에이전트 모드는 어떻게 작동하나요?

    Single: One-off tasks sent to a specific agent.
    Parallel: Dispatch multiple agents at once (max 8 tasks, 4 concurrent). Each runs independently.
    Chain: Sequential pipeline where each step receives the previous step's output via {previous} placeholder.
    All modes include cycle detection, depth limits (max 3), and concurrency control.

    단일: 특정 에이전트에 보내는 일회성 작업.
    병렬: 여러 에이전트를 한 번에 디스패치(최대 8개 작업, 4개 동시). 각각 독립적으로 실행.
    체인: 각 단계가 이전 단계의 출력을 {previous} 플레이스홀더로 받는 순차 파이프라인.
    모든 모드에 사이클 감지, 깊이 제한(최대 3), 동시성 제어가 포함됩니다.

    What does the slop-cleaner do? 슬롭 클리너는 무엇을 하나요?

    The slop-cleaner automatically runs after successful worker or plan-worker executions. It detects and removes LLM-specific code patterns (unnecessary comments, over-verbose logging, redundant type assertions, etc.) while preserving the original behavior.

    슬롭 클리너는 worker 또는 plan-worker 실행 성공 후 자동으로 실행됩니다. 원래 동작을 유지하면서 LLM 특정 코드 패턴(불필요한 주석, 과도한 로깅, 중복 타입 단언 등)을 감지하고 제거합니다.

    How does the FFF search engine change my workflow? FFF 검색 엔진은 워크플로를 어떻게 바꾸나요?

    It overrides the built-in find and grep tools with a native FFF-backed implementation and adds multi_grep. In both mode it also replaces @ file autocomplete suggestions. Use /fff-health to inspect engine status and /fff-mode tools-only if you only want the tool overrides.

    기본 find / grep 도구를 네이티브 FFF 기반 구현으로 교체하고 multi_grep를 추가합니다. both 모드에서는 @ 파일 자동완성도 대체합니다. 엔진 상태는 /fff-health로 확인하고, 도구 override만 원하면 /fff-mode tools-only를 사용하세요.

    Can I use this extension without the /ultraplan command? /ultraplan 명령 없이 이 익스텐션을 사용할 수 있나요?

    Absolutely. The three slash commands are independent. Use /clarify + /plan for single-session tasks. Use /ultraplan only when you need milestone-level decomposition for complex, multi-day projects.

    물론입니다. 세 개의 슬래시 명령은 독립적입니다. 단일 세션 작업에는 /clarify + /plan을 사용하세요. 복잡한 며칠짜리 프로젝트에 마일스톤 수준 분해가 필요할 때만 /ultraplan을 사용하세요.
